## MDX Template and Formatting Guide

This document demonstrates all the different formatting options and content types you can use when creating MDX documentation files.

## Basic Text Formatting

You can use **bold text** to emphasize important points, *italic text* for subtle emphasis, and `inline code` for technical terms or commands.

You can also combine formatting like ***bold and italic*** or **bold with `code`** inside.

## Lists and Organization

### Unordered Lists
- First item with basic text
- Second item with **bold text**
- Third item with `inline code`
- Nested items:
  - Sub-item one
  - Sub-item two
    - Deep nested item
- Back to main level

### Ordered Lists
1. First step in a process
2. Second step with **important** information
3. Third step with `technical details`
4. Numbered sub-steps:
   1. Sub-step A
   2. Sub-step B
   3. Sub-step C
5. Continue main sequence

### Task Lists (Checkboxes)
- [x] Completed task
- [x] Another completed item
- [ ] Pending task
- [ ] Future task
- [x] ~~Completed and crossed out~~

## Best Practices for MDX

### Content Organization
1. **Use clear headers** to structure your content
2. **Include a table of contents** for long documents
3. **Use consistent formatting** throughout
4. **Add examples** for complex procedures

### Writing Style
- Write in **active voice** when possible
- Use **bullet points** for lists of items
- Include **specific examples** rather than abstract concepts
- **Test all procedures** before publishing

### Technical Details
- Always include **version numbers** for software
- Provide **alternative methods** when possible
- Include **troubleshooting steps** for common issues
- Add **contact information** for additional help
